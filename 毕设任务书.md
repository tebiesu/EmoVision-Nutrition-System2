# 郑州轻工业大学

## 毕业设计（论文）任务书

**题目**  
基于情感识别与视觉分析的个人饮食健康智能助手

**专业**  
软件工程（新型平台软件）

**学号**  
*********

**姓名**  
******

**主要内容：**  
基于个人日常饮食健康管理与情绪状态感知需求，开发“基于情感识别与视觉分析的个人饮食健康智能助手”。系统采用前后端分离模式，Web端用户界面基于 Vue 前端框架设计，后端管理与业务服务基于 Spring Boot 框架开发。系统围绕“采集-识别-分析-建议-追踪”构建闭环，主要功能模块包括用户健康档案管理、饮食记录与图片上传、视觉分析与食物识别、情感识别、营养评估、个性化饮食建议、历史趋势分析与可视化展示等。课题重点参考多模态情感分析方法与图像膳食评估研究成果，在工程实现中采用可扩展接口接入云端模型服务，并结合营养素质量评价指标形成可解释建议。

**基本要求：**  
能够在社会、健康、安全、法律、文化以及环境等现实约束条件下进行系统设计；设计方案科学合理，论据可靠，分析与论证充分；能够依据设计方案完成系统实现、测试、评价与优化，具有一定工程应用价值。系统应具备较好的可用性与稳定性，核心功能完整，数据处理流程清晰，关键接口可追踪。论文撰写应结构严谨、层次清晰、语言通顺、技术用语准确，符合学校本科毕业设计（论文）规范要求。

**时间要求：**  
第1-2周：完成课题调研、文献检索、外文翻译与文献综述。  
第3周：完成需求分析、总体方案设计，撰写并提交开题报告，参加开题答辩。  
第4-6周：完成系统详细设计与核心功能开发（用户档案、饮食记录、视觉识别、情感识别、建议引擎）。  
第7-8周：完成前后端联调与期中检查，补充异常处理与日志追踪。  
第9-12周：完成系统测试、性能优化、数据统计与可视化模块，形成稳定演示版本。  
第13-15周：完成论文初稿、评阅、查重与修改完善，准备答辩材料。  
第16周：提交最终论文与相关附件，参加系、院二级答辩。

**思政要求：**  
在完成毕业设计项目过程中，能够理解并自觉履行软件工程师对公众健康、安全与隐私保护的社会责任。能够分析和评价系统在健康建议、数据采集、算法推荐等方面对社会、法律、伦理与文化的影响，强化安全合规意识与底线思维。项目实施中坚持学术诚信，杜绝抄袭与数据造假，体现严谨求实、精益求精和服务民生的工程职业素养。
